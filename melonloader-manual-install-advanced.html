Installing MelonLoader Manually

The automated installer is the easiest path for most people. This guide covers the cases where it isn't: non-Steam installs, Linux/Proton setups, or when you just want full control over what gets placed where.

When to go manual

  • Games running through Proton or Wine on Linux or macOS
  • Install locations the automated installer doesn't detect correctly
  • You want to inspect exactly what files MelonLoader adds before running the game

Method 1 — Manual zip install (Windows)

01

Download the right archive

From the MelonLoader releases page, grab the zip matching your game's architecture:

Game typeDownload
64-bitMelonLoader.x64.zip
32-bitMelonLoader.x86.zip

Not sure which your game is? Check the game's .exe properties, or just try one — if the game won't launch afterward, the other architecture is almost always the fix.

02

Extract into the game folder

With the game fully closed:

  1. Extract the MelonLoader folder from the zip directly into the game's install folder (same folder as the game's .exe)
  2. Extract version.dll and dobby.dll from the zip into that same folder
03

Launch once to verify

Start the game. The MelonLoader console window should appear alongside it. This first launch generates Mods, MelonLoader/Plugins, and UserData.

Method 2 & 3 — Linux and macOS

Both rely on running the game through a Windows compatibility layer. Pick your platform:

This assumes you already have the game running under Proton via Steam Play.

01

Set up protontricks

If you don't have it:

pipx install protontricks
02

Create a clean prefix, set Windows 10 mode

protontricks [appid] --gui

Use the GUI to configure the prefix, then set the Windows version to Windows 10 under the Wine configuration tab.

03

Install the .NET runtime

protontricks [appid] dotnetdesktop6

This installs the .NET 6.0 Desktop Runtime, which Il2Cpp games need to run MelonLoader.

04

Install MelonLoader itself

Either run the automated installer through the prefix:

protontricks-launch --appid [appid] MelonLoader.Installer.exe

...or follow the Method 1 manual zip steps above, extracting directly into the game's install folder inside the Proton prefix.

05

Launch through Steam as normal

If everything's set up correctly, the MelonLoader console should appear — you may need to alt-tab, since Proton sometimes launches it behind the game window.

The general approach mirrors Linux, run through CrossOver or a Wine-based launcher.

  1. Make sure you have a working install of the game via CrossOver or a Wine-based launcher
  2. Use winecfg (under the CrossOver/Wine Tools tab) and winetricks in the same prefix to install the required dependencies — same packages as the Linux steps
  3. Set the Windows version to 10 in winecfg, click OK, and close
  4. Install MelonLoader using either the automated installer or the manual zip method, run inside that same prefix

Some Wine-based tools (like Sikarugir-managed Steam installs) need extra steps to expose winecfg/winetricks — e.g. opening Configure.app from the app's package contents. Check your specific tool's docs for the exact path.

File reference — what goes where

File / FolderPurpose
version.dllHooks MelonLoader into the game on launch
dobby.dllRequired dependency for hooking
MelonLoader/Core loader files
MelonLoader/Plugins/Plugin .dll files (loaded before the game)
Mods/Mod .dll files (loaded after the game initializes)
UserData/Configs, save data, Loader.cfg

Loader.cfg lives at UserData/Loader.cfg and only appears after MelonLoader has run at least once:

[loader]
# Disables MelonLoader entirely — same as launching with --no-mods
disable = false

# Same as launching with --melonloader.debug
debug_mode = true

Troubleshooting

Installer doesn't detect the game (Windows)

Point it manually at the game's .exe rather than relying on auto-detect.

Crash on startup, any platform

Missing or broken .NET 6 Desktop Runtime. On Windows, install/repair it directly. On Linux, re-run the protontricks dotnetdesktop6 step.

Console window never appears

Check you extracted into the actual game folder — the one with the game's .exe, not a subfolder or the Steam library root. On Proton, try alt-tabbing.

Il2Cpp-specific mod errors on Linux

Usually a .NET runtime issue inside the Proton prefix — re-run the protontricks step rather than reinstalling MelonLoader itself.

Uninstalling

  1. Close the game
  2. Delete version.dll and dobby.dll from the game's install folder
  3. Delete the MelonLoader folder
  4. (Optional, full clean removal) also delete Mods, Plugins, and UserData

On Linux/macOS, these all live inside the game's folder within the Proton/Wine prefix — same steps, just navigate there first.
